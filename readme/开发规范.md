# flutter_luckin_coffee开发规范

## 类使用大驼峰

```
  class FlutterDemo {

  }
```

## 类的文件名和类名统一

```
class FlutterDemo {

}
```

文件名: FlutterDemo.dart

## 方法使用小驼峰

```
flutterDemo() {

}
```

## 一个方法不单独创建一个文件

## 重要的注释使用 "///" 如常量、类、方法

## 类注释

类注释写在构造函数或者工厂函数上，使用的时候会有提示

```
class FlutterDemo {
  final String name;

  /// 方法说明
  /// ```
  /// @param {String} name - 参数说明
  /// ```
  FlutterDemo(this.name);
}
```

## 方法注释同类注释

## 样式如果不会更改，使用const

``` dart
padding: const EdgeInsets.symmetric(horizontal: 15)
```
